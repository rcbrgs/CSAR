================================================================================
CLOCK MODEL NOTE
================================================================================

The AI has no independent clock. All elapsed-time detection is based on
prompt arrival. A trigger nominally set for "every hour" fires on the first
prompt after the threshold has passed — not at the exact moment the threshold
is crossed. This is a structural constraint of the architecture, not a bug.

Triggers must be specified as:
  "fires on first prompt after N minutes have elapsed"
not:
  "fires at time T"

All time-based triggers in this file follow this model.

Session time is estimated from session flow when not explicitly timestamped.
If elapsed time cannot be determined: skip silently and note "not verifiable."

================================================================================
TRIGGERS
================================================================================

TRIGGER ARRIVAL: SESSION FIRST PROMPT TIMESTAMP
  Trigger:  the first prompt of every session, exactly once.
  Output:   write a [SESSION] arrival annotation to the active
            write target:
            {
              'name':    "session_arrival_[YYYYMMDD]_[seq]",
              'facets':  "[SESSION] [E:arrival] [T:session_date]",
              'snippet': "Session started. First prompt arrived.",
            }
  Note:     reserved for future session-boundary behaviors.
            Does not gate TRIGGER CRON — CRON runs independently
            on every prompt.
  Fires:    silently. No output to prompter.

TRIGGER PROMPT: EVERY PROMPT TIMESTAMP
  Trigger:  every prompt without exception (passive gate).
  Output:   write a [SESSION] prompt annotation to the active
            write target:
            {
              'name':    "session_prompt_[YYYYMMDD]_[seq]",
              'facets':  "[SESSION] [E:prompt] [T:session_date]",
              'snippet': "Prompt arrived. [seq N in session.]",
            }
  Note:     this annotation is the universal timestamp. TRIGGER CRON
            reads the most recent [E:prompt] annotation as "now" for
            all interval calculations. The corpus measures its own
            time by writing.
  Fires:    silently before TRIGGER CRON on every prompt.

TRIGGER CRON: SCHEDULED ITEM DISPATCHER
  Trigger:  every prompt (passive gate), after TRIGGER PROMPT has
            written the current timestamp annotation.
  Process:
    STEP 1 — load the CRON LIST from the active Layer 2 write target.
             If no CRON LIST annotation exists: skip silently.
    STEP 2 — read "now" from the most recent [E:prompt] annotation
             timestamp in any Layer 2 file.
             If not resolvable: skip all items silently.
    STEP 3 — for each item in the CRON LIST:
               a. Compute elapsed = now − item's last_activated.
                  If last_activated is null: treat elapsed = ∞.
               b. If elapsed ≥ item's interval: ACTIVATE the item.
                  Execute item's tool or trigger. Update last_activated
                  to current prompt timestamp.
               c. If elapsed < interval: skip item silently.
    STEP 4 — write the updated CRON LIST back to the write target
             if any item was activated.
  Output:   activated items fire their own output (see each tool).
            CRON itself produces no additional output.

  CRON LIST ANNOTATION FORMAT:
    {
      'name':    "cron_list",
      'facets':  "[TOOL] [M:practice] [T:YYYY-MM-DDTHH:MM:SS+TZ]",
      'snippet': "CRON LIST:
                  [
                    {
                      'name':           'item label',
                      'tool':           'T16 / lsAll() / [trigger name]',
                      'interval':       '60min / 24h / 7d / ...',
                      'last_activated': 'YYYY-MM-DDTHH:MM:SS+TZ or null',
                      'next_due':       'YYYY-MM-DDTHH:MM:SS+TZ or null',
                    },
                    ...
                  ]",
    }

  INTERVAL FORMATS:
    Nmin   — N minutes (e.g. 60min, 45min)
    Nh     — N hours   (e.g. 1h, 24h)
    Nd     — N days    (e.g. 1d, 7d)
    prompt — every prompt (interval = 0, always fires)

  ADDING AN ITEM:
    Add an entry to the CRON LIST annotation snippet.
    Set last_activated: null to fire on next session.

  REMOVING AN ITEM:
    Delete the entry from the CRON LIST annotation snippet.
    Or set interval: disabled to suppress without deleting.

TRIGGER 4-CLOCK: HOURLY ELAPSED (since last prompt in session)
  Trigger:  first prompt received after ≥ 60 minutes have elapsed since
            the previous prompt in the current session.
            Time estimated from session flow if not explicitly timestamped.
  Output:   review context for hourly triggers. Execute any that match.

TRIGGER 16: REMIND EVERY HOUR
  Trigger:  first prompt after ≥ 60 minutes since T16 last fired in
            this session. At session start: fires after 60 minutes from
            session-start if determinable, or after the 4th prompt
            exchange if timestamp unavailable.
  Gate:     (a) session appears to be active work — not a single-exchange
                lookup or greeting.
            (b) T16 has not fired within the last 45 minutes in this session.
  Output:   execute ALL active items in the HOURLY REMINDER LIST,
            appended after the main response, one line each.
            Items are subordinate to the response — never interrupt.
            Do not explain, do not pad, do not repeat within 45 minutes.
  Suppress: if the current prompt is itself about the reminder topic.

  HOURLY REMINDER LIST:
    Items in this list fire together on every T16 execution.
    To add an item: file a [REMINDER] annotation in any Layer 2 write target.
    To disable an item: set status: inactive in its annotation.
    Items execute in INVENTORY order of the file they are filed in,
    then in annotation sequence order within that file.

    DEFAULT ACTIVE ITEMS: none. Items are added by operator annotation.

  REMINDER ANNOTATION FORMAT:
    {
      'name':    "reminder_[short_label]",
      'facets':  "[REMINDER] [M:practice] [T:YYYY-MM-DD]",
      'snippet': "[STATUS: active/inactive]
                  Output line: exact text to emit when T16 fires.
                  Context: why this reminder exists.",
    }
    [REMINDER] is a Ranganathan domain facet (defined in CSAR.txt).
    Reminders are filed in any active Layer 2 write target.

================================================================================
CLOCK STATE
================================================================================

The Clock module maintains the following session state (in memory only):
  last_t16_fired:    timestamp of last T16 execution this session.
  session_start:     estimated session start time.
  last_csar_prompt:  timestamp of most recent [E:session_summary] in Layer 2.

State resets on new session. State is estimated, not measured.
